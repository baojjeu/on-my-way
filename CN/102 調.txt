（一）產生廣播風暴的原因為於區域網路中，Switch 或 Bridge 連接的方式形成一迴路（Loop），當一
     主機發送一廣播封包（例如：ARP Request），交換器會將該封包複製並且 Flood 到其他設備
     （除了剛剛的主機外），而其他位於迴路上的網路設備也會執行該動作並且重複下去，如此一來早成
     廣播風暴，導致線路癱瘓，佔據網路頻寬。

二、
（一）防火牆和路由器之區別：
     1. 需求面：
        (1) 防火牆的產生是基於人們對安全性之需求，專注於該封包是否允許被通過，以及通過後是否
            會對網路造成危害。
        (2) 路由器的產生是基於為了將封包在不同的網路間選擇一條最佳路徑，並送到正確的目的地。
            其中需考量許多因素，如：傳輸頻寬、線路品質、延遲時間及經過節點數等。
     2. 功能面
        (1) 防火牆的功能目的為保證任何非允許的封包「不通」
        (2) 路由器的功能目的為保證網際網路以及資料「可通」
     3. 安全面
        (1) 防火牆的初始設定即可防治絕大多數的攻擊，配置較為簡單，且出錯率低
        (2) 路由器需要高級的應用層級之設定，才能防範攻擊，配置較為複雜且易出錯。
（二）網頁應用防火牆與封包過濾防火牆之區別：
     1. 服務層級
        (1) 網頁應用防火牆提供及處理較複雜的應用層之服務，如：使用者驗證功能
        (2) 封包過濾型防火牆提供及處理網路層之封包過濾之基本功能
     2. 目的層面
        (1) 網頁應用防火牆為了確保應用層協定的安全，其可過濾傳送資料的內容與命令，因此支援嚴謹
            的使用者驗證功能
        (2) 封包過濾型防火牆為了決定該封包是否允許通過，其依據定義好的存取規則，應用至每個流經
            的 IP 封包
（三）DMZ 之功用
     DMZ 是一種網路架構的佈置方案，常使用的架設方式為在不信任的外部網路（Internet）和可信任
     的內部網路（Intranet）之間，建立一個面向外部網路的物理或邏輯子網路，如此防範來自外部的
     入侵攻擊。

三、
（一）TTL (Time To Live)是 IP 協定中的一個標頭欄位，用來表示封包可以存活多久，長度為 8 個
     位元，範圍為 0~255，其作法為當封包每次經過一個 Router 時，其值便被減 1，當 TTL 為 0，
     則丟棄該封包，為了避免封包在網路上無限制地傳送。
（二）若封包的資料量太大，則傳遞的過程中需做分割，因此 Fragment Offset 欄位則表示目前該原始
     資料分段的位置，常與 Identification 和 Flag 欄位搭配使用。

四、以下針對伺服器程式之反覆（iterative）和同時（concurrent）進行說明：
（一）反覆（iterative）
     反覆的伺服器為一次指服務一個客戶端，設計較簡單，適用於持續時間較短的交易。然而，如果交易
     持續時間過長，則會影響整個伺服器的效能。（圖）
（二）同步（concurrent）
     同步型伺服器可處理多個使用者，當伺服器與使用者進行連線後，便會建立一個 child server，
     以進行接下來的交易，如此一來，其他使用者不必等待當前使用者的交易結束後才能進行交易請求。
     Source: https://goo.gl/4av5nV

五、
（一）VPN 全名為虛擬私人網路（Virtual Private Network），是一種常用於連線中、大型企業，或團
     體與團體間的私人網路的通訊方法，其透過公用的 Internet 來傳送 Intranet 的網路訊息，且利
     用已加密的通道協定（Tunneling Protocol）來達到訊息的保密、訊息準確性及傳送端認證的安全
     效果。
（二）以下分別說明兩者之差異：
     1. 以 IPSec 的 Transport Mode
        (1) 連接方式：端點對端點（Internet 介於兩主機間）
        (2) 封包結構：IP Header | ESP Header | TCP/UDP Header | Data Payload |
                    ESP Trailer | ESP Auth data
     2. 以 IPSec 的 Tunnel Mode
        (1) 連接方式：網段對網段（Internet 介於兩 Security Gateway 間）
        (2) 封包結構： New IP Header | ESP Header | Original IP Header |
                     TCP/UDP Header | Data Payload | ESP Trailer | ESP Auth data
     source: https://goo.gl/N0liRX
（三）以下分別說明兩者之差異：
     1. 以 IPSec 技術建立的 VPN
        (1) 服務層級：網路層
        (2) 優點：開放的工業標準，適用於 IPv4 及 IPv6
        (3) 缺點：認證標頭（Authentication Header）不提供機密性（Confidentiality）
     2. 以 SSL 技術建立的 VPN
        (1) 服務層級：傳輸層
        (2) 優點：瀏覽器支援且設定簡便
        (3) 缺點：並非所有應用程式都能支援

六、
（一）設計之角度及其理由
     因為雖然 TCP 傳輸協定為可靠且保證資料的正確性，但要確保達成以上之條件，必須經由許多的管理
     程序來加以控制，例如：三向交握（Three-way handshake）、流量控制、壅塞控制等。而這些管理
     的程序也會使一些頻寬被浪費掉，且因其連結導向特性，收送雙方必須持續在線，若處於不佳之網路環境，
     則會因不斷地連接、斷開、連接，因此造成傳輸效率不佳，不適用於現今之特定網際網路服務，如：
     影音串流。
（二）為何需要 UDP
     由於現今大部份 Internet 應用如：線上遊戲、影音串流等服務要求較高的即時性以及同步性，因此 UDP
     傳輸協定因應而生。UDP 藉由其無連結導向的特性，沒有 TCP 的三向交握（Three-way handshake）
     及流量管理等程序，因此能使傳輸效率達到最高。
